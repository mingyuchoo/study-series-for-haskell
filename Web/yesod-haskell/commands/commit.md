## 요구사항 추적 커밋 커맨드

이 커맨드는 CLAUDE.md의 "변경 추적성" 원칙에 따라 커밋 메시지에 관련 REQ-ID를 자동으로 태깅합니다.

### 실행 흐름

1. **변경 파일 확인**: `git status`와 `git diff`를 실행하여 staged 및 unstaged 변경 파일 목록을 수집합니다.

2. **변경 사항이 없는 경우**: "커밋할 변경 사항이 없습니다."를 출력하고 종료합니다.

3. **requirements.md 파일 읽기**: 프로젝트 루트의 `requirements.md`를 읽어 모든 요구사항 항목을 파싱합니다.
   - 각 행에서 REQ-ID, 제목, 영향 파일 목록을 추출합니다.

4. **REQ-ID 매핑**: 변경된 파일 각각에 대해 `requirements.md`의 "영향 파일" 컬럼과 비교합니다.
   - 파일 경로의 부분 일치도 허용합니다 (예: `src/Handler/Post.hs`는 `Handler/Post.hs`와도 매칭)
   - 매칭되는 REQ-ID를 모두 수집합니다 (중복 제거)

5. **매핑 결과 보고**: 사용자에게 다음을 보여줍니다:
   - 변경된 파일 목록
   - 매핑된 REQ-ID 목록과 각 요구사항의 제목
   - 매핑되지 않은 파일이 있으면 별도로 표시

6. **커밋 메시지 구성**:
   - 매핑된 REQ-ID가 **1개**인 경우: `[REQ-XXXX] <메시지>`
   - 매핑된 REQ-ID가 **여러 개**인 경우: `[REQ-XXXX][REQ-YYYY] <메시지>`
   - 매핑된 REQ-ID가 **없는 경우**: 사용자에게 REQ-ID를 직접 지정하도록 요청합니다. REQ-ID 없이는 커밋하지 않습니다.
   - `<메시지>` 부분:
     - 사용자가 인자로 커밋 메시지를 제공한 경우 그대로 사용
     - 인자가 없는 경우 변경 내용을 분석하여 적절한 커밋 메시지를 제안하고, 사용자에게 확인을 요청합니다

7. **스테이징 및 커밋**:
   - 아직 스테이징되지 않은 변경 파일이 있으면, 사용자에게 어떤 파일을 스테이징할지 확인합니다
   - 확인 후 `git add`로 파일을 스테이징합니다
   - `git commit -m "<커밋 메시지>"` 형식으로 커밋합니다
   - 커밋 메시지는 반드시 HEREDOC 형식을 사용합니다

8. **결과 출력**: 커밋 결과를 보여주고, 최근 커밋 로그를 출력합니다.

### 커밋 메시지 형식

```
[REQ-F001] 포스트 목록 조회 핸들러 추가
```

```
[REQ-F001][REQ-F002] 포스트 CRUD 및 인증 연동
```

### 중요 규칙

- REQ-ID가 매핑되지 않는 변경 사항은 사용자 확인 없이 커밋하지 않습니다
- `requirements.md` 자체의 변경은 REQ-ID 매핑에서 제외합니다 (메타 파일)
- `git push`는 수행하지 않습니다. 커밋만 생성합니다
- 커밋 메시지 끝에 `Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>`를 추가합니다
- 기존 커밋 스타일(최근 git log)을 참고하여 메시지 톤을 맞춥니다
