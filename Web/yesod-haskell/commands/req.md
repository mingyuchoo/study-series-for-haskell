## 요구사항 관리 커맨드

이 커맨드는 CLAUDE.md의 "변경 추적성" 원칙에 따라 `requirements.md`를 관리합니다.

### 실행 흐름

1. **requirements.md 확인**: 프로젝트 루트에 `requirements.md`가 있는지 확인합니다.

2. **없으면 초기 생성**: 아래 템플릿으로 `requirements.md`를 생성합니다:

```markdown
# 요구사항 관리 문서

| REQ-ID | 구분 | 제목 | 상태 | 영향 파일 | 등록일 |
|--------|------|------|------|-----------|--------|
```

1. **인자 분석**: 사용자가 입력한 인자를 분석합니다.

   - **인자가 없는 경우**: `requirements.md`의 현재 상태를 요약하여 보여줍니다.
     - 전체 요구사항 수, 상태별(등록/분석완료/진행중/완료) 개수
     - 진행중인 요구사항 목록

   - **`register <설명>` 또는 `등록 <설명>`**: 새 요구사항을 등록합니다.
     - 설명 내용을 분석하여 구분을 자동 판별합니다:
       - `F` (기능): 새로운 기능 추가
       - `N` (비기능): 성능, 보안, 유지보수 관련
       - `B` (버그): 버그 수정
       - `R` (리팩토링): 코드 개선
     - 기존 REQ-ID의 마지막 번호를 확인하여 다음 번호를 부여합니다
     - 형식: `REQ-{구분}{번호:3자리}` (예: REQ-F001, REQ-B002)
     - 상태: `등록`
     - 등록일: 오늘 날짜 (YYYY-MM-DD)
     - 사용자에게 "REQ-F003으로 등록했습니다." 형태로 안내합니다

   - **`status <REQ-ID> <상태>`**: 특정 요구사항의 상태를 변경합니다.
     - 허용되는 상태: `등록`, `분석완료`, `진행중`, `완료`, `보류`, `취소`
     - 상태 변경 후 사용자에게 결과를 안내합니다

   - **`files <REQ-ID> <파일목록>`**: 요구사항의 영향 파일을 갱신합니다.
     - 파일 목록은 쉼표로 구분하여 입력합니다

2. **결과 출력**: 변경된 `requirements.md`의 해당 행을 보여줍니다.

### 중요 규칙

- REQ-ID는 절대 중복되지 않아야 합니다
- 이미 `완료` 상태인 요구사항의 상태를 다시 `진행중`으로 변경할 때는 사용자에게 확인을 요청합니다
- `requirements.md` 파일은 증분 수정(Edit)만 허용합니다. 전체 재생성하지 않습니다
